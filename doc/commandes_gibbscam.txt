All Macro Commands

PERMANENT	<var name>, <var name>, <var name> (max 50 variables)
GLOBAL	<var name>, <var name>, <var name> (max 50 variables)
LOCAL	<var name>, <var name>, <var name> (max 50 variables)


Global variables can be used by any macro that is called in a macro session. Local variables can only be used in the macro where they are defined. Permanent variables are retained when the macro has finished. These values will be available to another macro that is run from the plugins menu.
ARGS	<var name>, <var name>, <var name> (max 50 variables)

Define the list of arguments passed to this macro when it is called by another macro. These variables will be local to this macro


IF	<value> <condition> <value> THEN

Use IF to check if values are the same or different, or if one value is greater than or less than another. Either add the command to be exectuted after the THEN keyword.

FOR	<variable> = <start> TO <end> [STEP <value>]
NEXT	<variable>
CONTINUE

Execute all of the commands between FOR and NEXT using the <variable> as a counter. The first time through the loop it will be set to the <start> value. When the NEXT command is reached the <variable> will be incremented by adding the STEP value (if no STEP value is programmed, the increment will be 1)  If the <variable> value is less than or equal to the <end> value, the macro will jump back to execute the command immediately after the FOR command. When the CONTINUE is used inside a FOR/NEXT loop it has the same effect as jumping straight to the NEXT command and incrementing the <variable>


GOTO	<label>
CALL	<macro name> [<arg 1>, <arg2>, <arg3> (max 10 arguments)]

Use the GOTO command to jump to a new line in the macro. The <label> is a text string, and the line to jump to is defined by inserting a line containing just the label, followed by a colon ':' Use the CALL command to call another macro. You may optionally pass up to 10 arguments (values) to the macro. The macro that is called must contain an ARGS command followed by a matching number of variable names to receive the values of each argument.

#INCLUDE <filename>
You can define commonly used sections of macro code in a separate file and
insert it using the #INCLUDE command. When the macro is executed this code is
treated as though it was already inside the macro file, so you must take that any
variables used match those already defined in the macro into which it is inserted.

When assigning a value to a variable, you can use expressions that contain
any of the the following internal functions.

ABS(n)   (get the absolute value the number 'n', positive value)
INT(n)   (get the integer part of the value 'n')
SQRT(n)  (get the square root of the number 'n')

SIN(a)   (sine of the angle 'a')
COS(a)   (cosine of the angle 'a')
TAN(a)   (tangent of the angle 'a')
ASIN(n)  (angle whose sine is the given value 'n')
ACOS(n)  (angle whose cosine is the given value 'n')
ATAN(n)  (angle whose tangent is the given value 'n')

LEN(a$)   (number of characters in the given string)
VAL(a$)   (value of a string)
ASC(a$)   (ascii code of a string variable, must be a single character)

LEFT$(a$,n)   (get the 'n' leftmost characters from the string)
RIGHT$(a$,n)  (get the 'n' rightmost characters from the string)
MID$(a$,p,n)  (get 'n' characters from the string, starting at the 'p'th character)

LTRIM$(a$)    (trim any leading spaces from the string)
RTRIM$(a$)    (trim any trailing spaces from the string)
TRIM$(a$)     (trim leading spaces and trailing from the string)

LCASE$(a$)    (convert a string to upper case)
UCASE$(a$)    (convert a string to lower case)
CHR$(n)       (get the character that represents the ascii value 'n')

ANGLE_LINE(x,y)   (get the angle of a line with the given X and Y components)

CONTOUR
CONTOUR_3D
START	<xs> <ys> [<zs>]
RAPID	<xe> <ye> [<ze>]
LINE	<xe> <ye> [<ze>]
ARC	<xc> <yc> <xe> <ye> [<ze>] <CW/CCW>

CONTOUR is used to define a 2D shape The first line after CONTOUR command must be an open square bracket ‘[‘. Then use the START command to specify the start position of the contour, followed by a series of LINE and/or ARC commands to define the end of each line/arc feature in the contour. After the final feature has been defined, add a line containing a close square bracket ‘]’. Where an XY position needs to be defined, you may replace this with a point number (eg P16). The arc center position may be replaced by a circle number (eg C21). Once the contour has been created, the variable ContourRef will be set to the number of the first feature in the contour. This may then be used to select the shape.

The HOLEFR commands are used to read data from the Hole Manager

This data is divided into groups. Use the HOLEFR command to load the data
and return the number of groups of data. Then use HOLEFR_GET_GROUP to see
how many hole features there are in specific group. Finally HOLEFR_GET_HOLE_DATA is
used to get the data for each hole feature in a given group.

HOLEFR_GET_DATA  <WG> <num groups>
HOLEFR_GET_GROUP <group> <num hole features>
HOLEFR_GET_HOLE  <hole num>

You must use HOLEFR_GET_DATA to load the data for a given WorkGroup
Then use HOLEFR_GET_GROUP to load the data for a given hole group
Then use HOLEFR_GET_HOLE to get data for each hole feature in the group
The hole feature data is set in a number of variables that you can use

HoleFrRef, HoleFrType, HoleFrCS, HoleFrRevCS, HoleFrThroughType
HoleFrDia, HoleFrDiaTop, HoleFrDepth, HoleFrMidDepth, HoleFrFullDepth

HoleFrError, which will be zero if successful

GET_COMMANDS <filename> [<option>] [<ver1> <ver2> <ver3>]

Get this list of commands. Use 1 for <option> to get the list
of commands only, use 2 to get the list plus some additional notes.

Use the version number to show commands added after a given version.
For example, GET_COMMANDS "commands.txt", 2, 9, 1, 3. This will get
commands and notes, for all commands added at version 9.1.3 or higher.

Set the version number to 0,0,0 to get all commands.
